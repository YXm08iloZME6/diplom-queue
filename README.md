# Электронная очередь

Система управления очередью для клиентских офисов обслуживания. **Терминал** самообслуживания позволяет посетителям получать талоны на услуги; **операторы** вызывают и обслуживают талоны у своих окон; **администраторы** управляют пользователями, услугами, окнами и настройками; публичные **табло** в реальном времени показывают вызванные и ожидающие талоны.

## Возможности

- **Терминал самообслуживания** — иерархический выбор услуг, форма ввода данных посетителя (ФИО, телефон), формирование талона (`буква + порядковый номер`) и упрощённый режим работы в одно нажатие.
- **Рабочее место оператора** — вызов следующего, повторный вызов, начало обслуживания, завершение, отмена и перенаправление талона.
- **Панель администратора** — управление пользователями и ролями, услугами (дерево), окнами, настройками системы, статистика и сброс очереди.
- **Главный дисплей** — табло обслуживаемых и ожидающих талонов в реальном времени со звуковым оповещением и анимацией выделения при вызове и повторном вызове.
- **Обновление в реальном времени** — изменения мгновенно доставляются всем подключённым клиентам через SignalR (без перезагрузки страницы).
- **REST API** — дополнительный интерфейс с авторизацией по JWT, предоставляющий часть операций; документирован через Swagger.

## Архитектура

Решение на .NET 6, построенное по **луковичной архитектуре (Onion Architecture)**. Зависимости направлены внутрь — к доменному ядру `Domain`.

| Проект | Назначение |
|---|---|
| **Domain** | Сущности (`Ticket`, `Service`, `User`, `Role`, `Window`, `Settings`) и перечисления. Ядро без зависимостей. |
| **Application** | DTO, интерфейсы репозиториев и сервисов, бизнес-логика. Зависит от Domain. |
| **Infrastructure** | `DbContext` EF Core, миграции, реализации репозиториев, SignalR-хаб и нотификатор. Зависит от Application и Domain. |
| **Web** | **Основное приложение** — ASP.NET Core MVC (Razor), cookie-аутентификация, хост SignalR, Tailwind CSS. Зависит от Infrastructure. |
| **Api** | Дополнительный REST API — авторизация по JWT, Swagger. Предоставляет часть сервисов. |

Приложения **Web** и **Api** — независимые хосты над одной базой данных PostgreSQL. **Web является основным продуктом**, Api реализован частично.

### Обмен данными в реальном времени

Сервисы вызывают события через `IQueueNotifier` → `QueueNotifier` рассылает клиентские методы (`NewTicket`, `UpdateTicket`, `CallTicket`, `RecallTicket`) через хаб `QueueHub` по адресу `/hubs/queue`. На стороне браузера используется клиент `@microsoft/signalr`.

## Технологии

- **Бэкенд:** C# / .NET 6, ASP.NET Core MVC и Web API
- **База данных:** PostgreSQL 16 + Entity Framework Core 6 (Npgsql, snake_case)
- **Реальное время:** SignalR
- **Аутентификация:** cookie (Web) / JWT (Api); пароли хешируются с помощью BCrypt
- **Фронтенд:** представления Razor, Tailwind CSS v4, JavaScript

## Требования

- [.NET 6 SDK](https://dotnet.microsoft.com/download/dotnet/6.0)
- [Docker](https://www.docker.com/) (для PostgreSQL)
- [Node.js / npm](https://nodejs.org/) (для Tailwind — сборка Web запускает его автоматически)

## Запуск

```bash
# 1. Запустить PostgreSQL (и pgAdmin)
docker compose up -d            # Postgres → localhost:5435, pgAdmin → localhost:8080

# 2. Применить миграции базы данных
dotnet ef database update --project Infrastructure --startup-project Web

# 3. Запустить основное веб-приложение
dotnet run --project Web        # https://localhost:7233  /  http://localhost:5129

```

Первая сборка автоматически генерирует CSS Tailwind и копирует клиент SignalR (цель MSBuild `Tailwind` в `Web.csproj`).
